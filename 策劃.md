# QuickBroadcast 技術策劃案（終版，供 AI 編程工具參考）

## 1. 產品功能概要
- **純廣播工具**：僅支持一對所有廣播，無在線列表、無點對點私聊。
- **發送端**：極簡視窗（暱稱欄、消息欄、發送按鈕），可常駐系統托盤，右鍵打開面板或退出。
- **接收端**：後臺靜默監聽，收到消息後在主顯示器右下角彈出無邊框置頂通知，顯示發送者與內容，8 秒後自動消失。
- **可靠性**：每條消息連續發送 3 次（間隔 20ms），接收方通過 UUID 去重。
- **跨平臺**：Windows、macOS、Linux。

## 2. 技術棧
- **語言**：C++23
- **GUI**：FLTK 1.4.x（靜態鏈接，最終可執行體積 < 5 MB）
- **異步網絡**：asio（獨立版，header-only，使用 C++20 協程）
- **JSON 序列化**：reflectcpp （編譯期反射，輕量快速）
- **編碼轉換**：EatiEssentials 庫（提供任意編碼與 UTF-8 之間的轉換，保證不同地區系統發送中文不亂碼）
- **構建系統**：CMake
- **包管理**：優先使用 vcpkg；vcpkg 未提供的（如 FLTK 1.4）使用 FetchContent，且關閉測試、文檔等開關，並設置 `CMP0077` 爲 `NEW`

## 3. 架構設計

### 3.1 線程模型：單線程異步
- **無額外 IO 線程**：整個應用僅主線程一條線程。
- FLTK 事件循環主循環，asio 的 `io_context` 不調用 `run()`（阻塞），而是通過以下兩種方式手動推進：
  - 使用 `Fl::add_fd` 監聽 UDP socket 可讀事件，回調中調用 `io_context.poll()` 執行就緒協程。
  - 輔助使用 `Fl::add_timeout` 定期調用 `io_context.poll()`，處理定時器事件（如重發延遲、定時清理等）。
- 所有協程均在主線程中喚醒和執行，**無需 `Fl::awake`、無需互斥鎖、無需線程安全橋接**。

### 3.2 核心模塊
- `NetworkFunctionality`：提供 UDP socket 生命週期、協程接收循環與 `broadcast()` 協程接口，供 `main` 組合使用。
- `MessageHandler`：解析 JSON，UUID 去重（LRU 集合），觸發彈窗。
- `NotifyWindow`：無邊框、置頂、右下角定位的通知彈窗。
- `SendPanel`：暱稱與消息輸入、發送按鈕。
- `TrayFunctionality`：提供系統托盤圖標與菜單能力，供 `main` 組合使用。
- `Encoder`：封裝 EatiEssentials，實現本地編碼與 UTF-8 互轉。

### 3.3 事件驅動示意
```
FLTK 事件循環
 ├─ FL_READ (UDP socket) → on_socket_readable() → io.poll()
 ├─ Fl::add_timeout(定時器) → poll_asio() → io.poll()
 └─ FLTK UI 事件 → 按鈕回調 → 構造 BroadcastMessage → co_spawn broadcast()
```
所有代碼在主線程中順序執行，不存在競爭。

## 4. 編碼規範
- 遵循現代 C++ 慣用法與 C++ Core Guidelines
- 使用預編譯頭（`stdafx.h` / `stdafx.cpp`），這是唯一使用 .h .cpp 的 C++ 代碼文件。
- 其他代碼文件後綴：`.cxx` / `.hxx`
- 能用引用就用引用，必須用指針時才用指針
- 使用 `std::string_view`、`std::optional`
- 如果類或結構體中包含一個不會為空的指針，請使用 `gsl::not_null<T*>`。
- 如果 gsl::not_null<T*> 不期望在运行时改变，请使用 gsl::not_null<T const*>。
- 采行 EATI C++。
    - 用 Box 和 Rc 来管理内存，而非 std::unique_ptr 和 std::shared_ptr。举例：
    ```cpp
    // 采取：
    auto window = newBox(MyWindow{ arg1, arg2, arg3 });
    // 而不是：
    auto window = std::make_unique<MyWindow>(arg1, arg2, arg3);
    ```
    - 需要在头文件中写出 Box 的类型名时应使用 ESSM::Box 而不是 Essentials::Memory::Box。
    - 拒絕二段式初始化。盡量讓類創建即建立不變式。
    - 對於純粹的「數據組合」，應該使用結構體而非類，并且使用零之法則，即讓編譯器自動生成構造、移動等等函數。
    - 指針總是不可信的。在一個函數内，如果之前沒有檢查有效性，需要使用 UNWRAP 或 EXCEPT 宏來「解包」來使用。如：
    ```cpp
    // win 不期望爲空，爲空是一個編碼 BUG。
    // 如果 win 爲空，會顯示一個明顯的錯誤信息，並結束程序，而不是引發未定義行爲。
    UNWRAP(win).show();

    ```

### 4.1 編碼风格规范
- 指針（*）和引用（&）符號靠右，這是爲了延續慣例。
    ```cpp
    // 舉例
    void func(int *p, int &r);
    ```
- 类、结构体、命名空间采取帕斯卡命名法（PascalCase）。函数、变量皆采取驼峰命名法（camelCase）。
- 类、结构体、命名空间的定义，开大括号位于新行。此外，各种语句（if, for, while 等等）后的开大括号、函数定义后的开大括号皆不换行。大括号初始化列表是一个例外，它根据美观考虑是否换行。举例：
    ```cpp
    class MyWidget
    {
        int foo;
    };

    void testFunc() {
        while (true) {
            if (true) {
            } else {
            }
            try {
            } catch (const std::exception &e) {
            }
        }
    }
    ```
- 初始化语法的使用规则：原则上只用等号初始化和大括号初始化。
    - 如果被初始化对象在语义上与用以初始化的值是相似的，使用等号初始化。如：
    ```cpp
    // 语义上表现：让 a 等于 10
    int a = 10;
    // 语义上表现：让 str 等于 "hello"
    std::string str = "hello";
    ```
    - 如果被初始化对象在语义上与用以初始化的值是不同的，需要表达出「初始化对象利用被初始化对象初始化，但初始化对象与被初始化对象是两种截然不同的事物」，或者需要用多个参数来初始化，建议使用大括号初始化。如：
    ```cpp
    // 语义上表现：MyWidget 是一个窗口类，而 "MainWindow" 是一个字符串，它们是两种截然不同的事物。
    MyWidget widget{ "MainWindow" };
    // 多个参数初始化 GameObject
    GameObject obj{ "GUIRoot", { 100, 152 } };
    ```
    - 如果利用一個函數的返回值初始化變量，應該使用 auto。
    ```cpp
    auto sth = makeSth();
    auto &ref = getRef();
    ```
    - 類的構造函數的成員初始化列表，總是使用大括號初始化。
    ```cpp
    MyWidget::MyWidget()
        : member1{ sth },
          member2{ sth } { }
    ```

## 6. 協議與端口
- **端口**：`54321`（UDP）
- **消息格式**（JSON）：
```json
{
  "type": "broadcast",
  "id": "uuid-string",
  "name": "用戶自定義暱稱",
  "host": "主機名",
  "content": "純文本消息內容",
  "ts": 1234567890
}
```
- **發送方式**：UDP 廣播至 `255.255.255.255`
- **可靠性**：每條消息連續發送 3 次，間隔 20ms

## 7. 關鍵實現細節

### 7.1 主線程 asio 推進
```cpp
// 獲取 socket 的原生句柄
int sock_fd = sock.native_handle();

// 可讀回調
void on_socket_readable(int fd, void*) {
    io.poll(); // 執行所有就緒的協程續程
}

int main() {
    // 設置監聽
    Fl::add_fd(sock_fd, FL_READ, on_socket_readable);
    // 可選的定時器兜底
    Fl::add_timeout(0.02, [](void*) {
        io.poll();
        Fl::repeat_timeout(0.02);
    });

    // 啓動協程
    asio::co_spawn(io, receive_loop(sock, handler), asio::detached);

    return Fl::run();
}
```

### 7.2 接收協程
```cpp
asio::awaitable<void> receive_loop(udp::socket& socket, MessageHandler& handler) {
    char buf[2048];
    udp::endpoint sender;
    for (;;) {
        auto [ec, len] = co_await socket.async_receive_from(asio::buffer(buf), sender,
            asio::as_tuple(asio::use_awaitable));
        if (ec) continue;
        handler.process(std::string_view(buf, len));
    }
}
```

### 7.3 發送協程（帶重複）
```cpp
asio::awaitable<void> broadcast(udp::socket& socket, const std::string& json) {
    udp::endpoint ep(asio::ip::address_v4::broadcast(), PORT);
    for (int i = 0; i < 3; ++i) {
        co_await socket.async_send_to(asio::buffer(json), ep, asio::use_awaitable);
        if (i < 2) {
            asio::steady_timer t(socket.get_executor(), std::chrono::milliseconds(20));
            co_await t.async_wait(asio::use_awaitable);
        }
    }
}
```

### 7.4 發送面板
- 暱稱輸入框 (`Fl_Input`)，消息輸入框 (`Fl_Multiline_Input` 或 `Fl_Input`)，發送按鈕。
- 發送回調：
```cpp
void on_send(Fl_Widget*, void* userdata) {
    auto* panel = static_cast<SendPanel*>(userdata);
    BroadcastMessage msg;
    msg.id = generate_uuid();
    msg.name = to_utf8(panel->name_input->value());   // 調用 Encoder
    msg.content = to_utf8(panel->message_input->value());
    msg.host = asio::ip::host_name();
    msg.ts = std::chrono::system_clock::now().time_since_epoch().count();
    std::string json = rfl::json::write(msg);
    // 在主線程發起異步廣播（協同）
    asio::co_spawn(panel->io, broadcast(panel->socket, json), asio::detached);
}
```


### 7.5 托盤管理
- 使用 FLTK 的 `Fl_Sys_Menu_Bar` 或直接調用各平臺 API 實現系統托盤圖標。
- 右鍵菜單選項：「打開發送面板」、「退出」。
- 面板關閉時不退出程序，僅隱藏，可從托盤重新打開。

## 8. CMake 配置要點
```cmake
cmake_minimum_required(VERSION 3.20)
project(QuickBroadcast LANGUAGES CXX)
set(CMAKE_CXX_STANDARD 23)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# 預編譯頭
target_precompile_headers(QuickBroadcast PRIVATE src/stdafx.hxx)

# vcpkg 依賴
find_package(asio CONFIG REQUIRED)
find_package(reflectcpp CONFIG REQUIRED)
find_package(EatiEssentials CONFIG REQUIRED)

# FLTK 1.4 通過 FetchContent
include(FetchContent)
cmake_policy(SET CMP0077 NEW) # 允許覆蓋子項目的 option()
FetchContent_Declare(fltk
  GIT_REPOSITORY https://github.com/fltk/fltk.git
  GIT_TAG release-1.4.0
)
set(FLTK_BUILD_TEST OFF)
set(FLTK_BUILD_EXAMPLES OFF)
set(FLTK_BUILD_SHARED_LIBS OFF)
FetchContent_MakeAvailable(fltk)

target_link_libraries(QuickBroadcast PRIVATE
  asio::asio
  reflectcpp::reflectcpp
  EatiEssentials::EatiEssentials
  fltk::fltk
)
```

## 9. 附加說明
- 程序啓動時檢查端口是否可用，不可用則彈出錯誤窗口並退出。
- UUID 生成使用 stduuid。
- 通知窗口使用 `Fl_Window` 派生，`clear_border()`，平臺特定代碼設置始終置頂，定時器 `Fl::add_timeout(8.0, ...)` 自動關閉。
- 所有代碼均爲單線程，無需考慮線程安全，極大簡化開發與調試。
